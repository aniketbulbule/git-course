5Ô∏è‚É£ BRANCHES
------------
Command to create branch:

Command to switch branch:

Meaning:
Branch is a separate workspace to work on features without affecting main code.
Safe place to experiment.

Merge back to main:

git merge feature-login

--------------------------------------------------------

6Ô∏è‚É£ MAIN BRANCH
---------------
Meaning:
Default branch in modern Git.
Holds stable/final code.
Command to rename current branch to main:

--------------------------------------------------------

7Ô∏è‚É£ MASTER BRANCH
----------------
Meaning:
Old default branch name.
Same as main, but outdated.
Not recommended now.

--------------------------------------------------------

8Ô∏è‚É£ REMOTE REPOSITORY
-------------------
Meaning:
Remote is the online copy of your project (GitHub, GitLab, Bitbucket).
Used to push, fetch, and share code.

Command to add remote:

Check remote URL:

--------------------------------------------------------

9Ô∏è‚É£ ORIGIN
----------
Meaning:
Origin is the nickname for remote repository.
Default name used in GitHub workflows.
Used when pushing or pulling code.

Example:
