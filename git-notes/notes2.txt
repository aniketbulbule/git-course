=====================================
WHAT IS CLONE IN GIT? (IN MY WORDS)
=====================================

Clone means creating a full copy
of a remote repository (GitHub/GitLab/Bitbucket)
into my local system.

When I clone:
- I get all files
- all commits
- all branches
- complete history


=====================================
BASIC CLONE COMMAND
=====================================

Command:
git clone <repository-url>

Use:
- Downloads repository from remote
- Creates a new folder automatically
- Connects repo with remote origin


Example:
git clone https://github.com/user/project.git


=====================================
CLONE INTO SPECIFIC FOLDER
=====================================

Command:
git clone <url> folder_name

Use:
- Clones repo into given folder name


Example:
git clone https://github.com/user/project.git myProject


=====================================
CLONE USING SSH
=====================================

Command:
git clone git@github.com:user/project.git

Use:
- Secure authentication
- No password required every time


=====================================
SHALLOW CLONE
=====================================

Command:
git clone --depth 1 <url>

Use:
- Downloads only latest commit
- Faster clone
- Saves space


=====================================
CLONE SPECIFIC BRANCH
=====================================

Command:
git clone -b branch_name <url>

Use:
- Clones only selected branch


Example:
git clone -b dev https://github.com/user/project.git


=====================================
CLONE WITHOUT CHECKOUT
=====================================

Command:
git clone --no-checkout <url>

Use:
- Clones repo without switching to files
- Used in advanced cases


=====================================
INTERVIEW ONE-LINERS
=====================================

Clone creates a local copy of a remote repository.

git clone downloads complete project with history.

Origin is automatically set after clone.



========================================================
GIT REMOTE, ORIGIN, MAIN, MASTER ‚Äì ALL IN ONE FILE
========================================================

COMMAND 1
---------
git remote -v

Meaning (in my words):
This command shows the remote repositories connected to my local Git project.

- -v ‚Üí verbose (shows URL clearly)
- Shows fetch and push URLs

Example output:
origin  https://github.com/user/repo.git (fetch)
origin  https://github.com/user/repo.git (push)

Use:
Used to check whether my project is connected to GitHub or not.


--------------------------------------------------------

WHAT IS REMOTE?
---------------

Meaning:
Remote means a Git repository that is stored on the internet (GitHub, GitLab, Bitbucket).

In simple words:
Remote is the online place where my project code is stored.

Use:
- To push code online
- To share code with others
- To backup project

Example command:
git remote add origin <repo-url>


--------------------------------------------------------

WHAT IS ORIGIN?
---------------

Meaning:
Origin is the default name given to the remote repository.

In simple words:
Origin is just a nickname for my GitHub repository URL.

Important point:
- "origin" is NOT a command
- It is just a name
- We can rename it, but mostly we use "origin"

Use:
Used while pushing or pulling code.

Example:
git push origin main


--------------------------------------------------------

WHAT IS MAIN?
-------------

Meaning:
Main is the default branch name in modern Git.

In simple words:
Main is the primary branch where final and stable code is kept.

Use:
- Production-ready code
- Final version of project

Example:
git branch -M main


--------------------------------------------------------

WHAT IS MASTER?
---------------

Meaning:
Master was the old default branch name.

In simple words:
Master is the same as main, but older Git versions used it.

Important:
- GitHub now uses "main"
- "master" is still valid but not recommended

Example:
git branch -M main


--------------------------------------------------------

MAIN vs MASTER
--------------

Main:
- New default branch
- Recommended
- Used by GitHub

Master:
- Old default branch
- Still works
- Not preferred now


--------------------------------------------------------

WHAT IS BRANCH?
---------------

Meaning:
Branch is a separate line of development.

In simple words:
Branch allows me to work on features without disturbing main code.

Use:
- Feature development
- Bug fixing
- Safe experimentation

Example:
git branch feature-login


--------------------------------------------------------

COMPLETE SERIES (REAL FLOW)
---------------------------

1. git init
   ‚Üí Start Git in project

2. git remote add origin <repo-url>
   ‚Üí Connect GitHub

3. git remote -v
   ‚Üí Verify remote connection

4. git branch -M main
   ‚Üí Set default branch

5. git push -u origin main
   ‚Üí Push code to GitHub


--------------------------------------------------------

ONE-LINE INTERVIEW SUMMARY
-------------------------

- Remote ‚Üí Online repository
- Origin ‚Üí Nickname of remote
- Main   ‚Üí Default working branch
- Master ‚Üí Old default branch
- git remote -v ‚Üí Shows remote URL details
